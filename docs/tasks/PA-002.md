# Task Spec: PA-002 Project Lifecycle + File Format

## 1. Assignment

- **Sub-project:** `packages/power-app`
- **Branch:** `feat/PA-002-project-lifecycle`
- **Depends on:** PA-001 (scaffold must be complete)
- **Blocked by:** None

## 2. Objective

Implement project lifecycle (new, open, save, save-as) and the `.eunoistoria` file format (magic header + obfuscated zip). Users can create new projects, open existing `.eunoistoria` files, and export their work. Working directory persists across sessions (game engine paradigm).

## 3. Behavior

### Project Lifecycle

- **On app launch:** Read `app.getPath('userData')/state.json` to get last active project uuid. If the working directory exists, resume immediately (no prompt). Otherwise, show empty "New/Open" screen.
- **New project:** Dialog prompts for project name. Create working directory `projects/<uuid>/`, initialize SQLite with migrations, create project record in db. Set as active.
- **Open file:** File picker → user selects `.eunoistoria` file. Decode file → extract sqlite → copy to new `projects/<uuid>/`. Set as active. If current project has unsaved work, prompt "Save before opening?"
- **Save (Cmd+S):** If `sourcePath` is set (project was opened from a file), export to that path. If `sourcePath` is null (new project), behave as Save As.
- **Save As:** File dialog → user picks location. Export working db to `.eunoistoria` → update `sourcePath`.
- **Close/Exit:** If `isDirty` (working dir has changes not exported), show warning. But don't block exit (work is always saved in working dir).

### File Format (`.eunoistoria`)

```
Bytes 0–11:   ASCII "EUNOISTORIA\0"  (12 bytes, magic header)
Bytes 12–13:  Version u16 LE (0x0001)
Bytes 14–15:  Reserved u16 LE (0x0000)
Bytes 16+:    ZIP archive, first 4 bytes XOR'd with 0x42
```

- **Open:** Read file → validate magic → version check → XOR bytes 16-19 back → extract zip → copy `db.sqlite` to working dir → init engine.
- **Save/Export:** Copy working `db.sqlite` → zip it → XOR first 4 bytes → prepend 16-byte header → write to disk.

### State Persistence

- `app.getPath('userData')/state.json` tracks: `lastActiveProjectUuid: string | null`, `projectMetadata: Map<uuid, { sourcePath, isDirty, name, createdAt }>`
- `isDirty` set to true on every document write, reset to false on save.
- Working directory: `app.getPath('userData')/projects/<uuid>/db.sqlite`

## 5. Test Cases

1. **New project** — user enters name, project is created and active, working dir exists, app shows project name.
2. **Open .eunoistoria** — user picks file, it's decoded, extracted, working dir populated, app resumes.
3. **Save with sourcePath** — project has sourcePath set, pressing Cmd+S overwrites the original file.
4. **Save as (null sourcePath)** — new project with no source file, Cmd+S opens Save As dialog, creates `.eunoistoria`.
5. **Prompt on new/open** — if current project is dirty, switching projects prompts "Save before opening/creating?" (Save / Don't Save / Cancel).
6. **Resume on launch** — if working dir exists and is valid, app resumes without prompt.
7. **File format integrity** — exported `.eunoistoria` can be reopened and data is identical.
8. **Magic header validation** — opening file with corrupted magic bytes shows error message (not crash).

- `unzipper` — extract zip files

**Error handling:**
- File not found → show error message
- Corrupted zip → show error message
- Invalid magic header → show error message
- Database initialization fails → show error message
- Disk full on save → show error message
