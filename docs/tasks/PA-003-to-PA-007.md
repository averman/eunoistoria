# Task Specs: PA-003 through PA-007 (Detailed Breakdown)

---

## PA-003: OmniSearch Sidebar

**Objective:** Implement left sidebar with omnisearch (title + alias). Users search documents, click result to open.

**Behavior:**
- Left sidebar is collapsible (toggle button in top-left)
- Search input field with placeholder "Search documents..."
- As user types, call `window.eunoistoria.documents.search(query)` (debounced 300ms)
- Display results as a scrollable list (title + alias preview, icon indicating leaf/composition)
- Click result → open document in main area
- Clear button resets search, shows empty state

**Test Cases:**
1. Search finds documents by title (case-insensitive)
2. Search finds documents by alias
3. Results update as user types (debounced)
4. Clicking result opens document
5. Search respects current project (doesn't mix projects)

**Files:**
- `src/main/ipc/documents.ts` — implement `documents.search(query)` handler
- `src/renderer/src/components/search/OmniSearch.tsx` — search UI component
- `src/renderer/src/store/search-store.ts` — Zustand search state
- `src/renderer/src/components/layout/LeftSidebar.tsx` — collapsible sidebar container

**Dependencies:** PA-FIX (adapter titleContains filter), PA-001, PA-002

## PA-004: Leaf Editor (CodeMirror)

**Objective:** CodeMirror 6 markdown editor with autosave and inline title editing.

**Behavior:**
- When a leaf document is selected, show CodeMirror editor in main area
- Document title is editable inline (above editor)
- Content has syntax highlighting for markdown
- Autosave on content change (debounced 500ms)
- Show visual indicator when saving (subtle "saving..." text)
- Keyboard shortcut: Cmd+S to save immediately

**Test Cases:**
1. Editor displays content for leaf documents
2. Typing content triggers autosave after 500ms
3. Title editing saves on blur
4. Cmd+S saves immediately
5. Switching documents saves pending changes

**Files:**
- `src/renderer/src/components/editor/LeafEditor.tsx` — CodeMirror component
- `src/renderer/src/store/editor-store.ts` — editor state (document, content, isDirty)
- `src/renderer/src/App.tsx` — route to LeafEditor when leaf selected

**Dependencies:** PA-001, PA-002

## PA-005: Composition Canvas

**Objective:** Visual workflow builder for compositions. Each item is rendered markdown with [+] buttons to add, preview toggle, variant selector, delete button.

**Behavior:**
- When a composition is selected, show canvas in main area
- Canvas displays slots vertically, each showing resolved markdown preview
- [+] buttons above, between, and below slots
- Each slot item shows: [◉ preview toggle] [variant selector (if group)] [×]
- Resolve button at bottom → calls `compositions.resolve` → shows markdown in read-only panel
- Up/down arrow buttons on each slot to reorder (for MVP, full drag-drop deferred)

**Test Cases:**
1. Composition renders slots as markdown previews
2. [+] button opens picker dialog to add document/variant group
3. Delete button removes slot
4. Up/down arrows reorder slots
5. Resolve button produces flat markdown

**Files:**
- `src/renderer/src/components/editor/CompositionCanvas.tsx` — main canvas component
- `src/renderer/src/components/shared/SlotItem.tsx` — individual slot rendering
- `src/renderer/src/components/shared/AddSlotButton.tsx` — [+] button component
- `src/main/ipc/compositions.ts` — IPC handlers (addSlot, removeSlot, reorderSlots, resolve)

**Dependencies:** PA-001, PA-002

## PA-006: Variant Groups + Selector

**Objective:** Variant group management and visual selection popup.

**Behavior:**
- Users can create variant groups via "New → Variant Group" menu
- Group shows as a selectable entity (in omnisearch, in picker dialogs)
- Slot item pointing to variant group shows group name + dropdown with current selection
- Clicking dropdown opens Radix Popover listing all members by name
- Click member name → selection updates → composition re-renders

**Test Cases:**
1. Create variant group
2. Add documents as members
3. Reorder members (position 0 = default)
4. Slot shows current variant selection
5. Popover lists all members
6. Clicking member updates resolution

**Files:**
- `src/main/ipc/variant-groups.ts` — IPC handlers (list, create, addMember, removeMember, reorderMembers, getMembers)
- `src/renderer/src/components/variant/VariantPopup.tsx` — Radix Popover variant selector
- `src/renderer/src/store/editor-store.ts` — extend with selectedVariants map

**Dependencies:** PA-001, PA-002, PA-005

## PA-007: End-to-End Validation + POWER_APP.md

**Objective:** Validate all features work together. Write POWER_APP.md sub-project spec.

**Behavior:**
- User creates new project
- Adds 2-3 leaf documents with content
- Creates composition with these documents as direct slots
- Creates variant group with 2 documents
- Adds variant group to composition
- Opens variant selector, switches between variants
- Clicks resolve → sees flat markdown output with selected variants
- Saves project → file dialog → creates `.eunoistoria`
- Closes app → reopens
- Clicks File → Open → picks `.eunoistoria` → project data is intact

**Test Cases:**
1. Full workflow: create → author → compose → variant → resolve → save → reopen
2. All UI elements are responsive and labeled correctly
3. No errors in console
4. Performance: resolve completes within 1 second

**Files:**
- `POWER_APP.md` — sub-project spec (manifest, public API, file structure, test strategy)
- `docs/tasks/PA-001.md` through `PA-006.md` — (ensure all task specs are present and accurate)
- Minor fixes across all PA modules (any bugs found during validation)

**Deliverable:**
- Working MVP: users can create, author, compose, use variants, save, and reopen projects
- Complete POWER_APP.md with accurate file manifest and test results

**Dependencies:** PA-001 through PA-006
